# Roadmap

## Public Readiness

- Keep README, install guidance, and release references aligned with the packaged CLI version
- Publish repeatable quickstarts for single-repo and multi-repo adoption
- Establish visible issue and release closure discipline

## Maintainer Automation

- Strengthen PR triage, review assistance, changelog drafting, and release-note generation
- Expand auditable upgrade workflows across AAA repositories
- Improve machine-readable outputs for automation and reporting

## Governance Hardening

- Tighten readiness, permission, and evidence-promotion gates
- Expand negative tests around runbook execution and package boundaries
- Continue reducing prompt-only behavior in favor of deterministic paths

## Ecosystem Adoption

- Publish additional sanitized dogfooding case studies
- Improve starter templates and public onboarding assets
- Gather external trial feedback and convert it into public issues and roadmap items
