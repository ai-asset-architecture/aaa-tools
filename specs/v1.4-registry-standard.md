# Specification: aaa-policies Registry Standard (Zone Zero)

## 1. Directory Structure
The `aaa-policies` repository (or directory) acts as the Single Source of Truth for distributed governance policies.

```
aaa-policies/
├── policies.json          # Registry Index (Manifest)
├── policies/
│   ├── safety/            # Policy ID: safety
│   │   ├── v1.0.0/
│   │   │   ├── policy.yaml      # Source DSL
│   │   │   └── check_safety.py  # Compiled Script (Signed/Hashed)
│   │   └── v1.1.0/
│   └── security/          # Policy ID: security
│       └── ...
└── README.md
```

## 2. Manifest Schema (`policies.json`)
The manifest must be a JSON file generated by `aaa/distribution/manifest.py`. It MUST adhere to the **Diamond Standard** (Schema Rigor).

```json
{
  "last_updated": "2026-01-28T12:00:00Z",
  "registry_version": "1.0",
  "policies": {
    "safety": {
      "versions": {
        "1.0.0": {
          "url": "policies/safety/v1.0.0/check_safety.py",
          "hash": "sha256:...",
          "dependencies": []
        }
      },
      "latest": "1.0.0"
    }
  }
}
```
